<h2>Структура проекта</h2>
В папке src/ лежат подпапки по модулям: N/, Z/, Q/, P/. В каждой из них уже лежит файл со структурой (numbN.h, numbZ.h и т.д.)(файл со структурой нужно использовать, чтобы записать число). Ваше задание — создать в соответствующей папке два файла: название_модуля.h и название_модуля.c. Вам не нужно реализовывать main.c, только функции, которые закрепелены за вами. Если вас не устраивает распределение функций, следует обращаться к архитектору, он рассмотрит еще раз распределение.<br>
<br>

    
<h2>Разработка</h2>

<h3>Модули</h3>
После того как вы получили задание, вы должны работать в своем ветвление, в котором вы будете делать коммиты независимо, но главное прежде, чем сделать его, проверьте работоспособность вашего кода, протестируйте его локально, используя заглушки.<br>
После того как вами были успешно написаны задания, делайте PULL REQUEST, код будет проверен, если он будет работать, он будет принят, иначе вы будете дорабатывать его. Если ваш пулреквест долго не принимается (из-за сжатых сроков более суток считается долгим ожиданием, тегайте в чате в тг).
<h3>Оформление кода:</h3>

<h4>1) Функции:</h4>
<br>
Не пишите длинные фукнции, не надо весь модуль писать в одной функцию, разбейте его на функции поменьше. Называйте переменные осмысленно, иначе работа будет возвращаться к автору на доработку.<br>


<h4>2) Имена функций</h4>
Имена функций должны начинаться с маленькой буквы, каждое последующее слово в названии выделяется заглавной буквой!<br>
Имена функций должны начинаться с глаголов:

```c
int getLength() {...}
```

или

```c
int isNumber() {...}
```

<h4>3) Комментарии:</h4>
В начале каждой функции, которую будете писать укажите себя 

```c
/*
  Автор: Иванов Иван
  Группа: 5388
/*
```

Шаблон:

```c
/*
  Имя или краткое описание объекта

  Развернутое описание

  Параметры: 
    1) Значение и что оно означает
    2) Значение и что оно означает

  Что возвращает функция тип_данных
*/
```

Пример:

```c
/*
  get_length_str

  Возвращает длину переданной строки

  Параметры: 
    1) char* string - строка, длину которой нужно посчитать
    
  Функция возращает длину строки - int
*/
```

<h4>4) Заглушки</h4>
Как вы могли догадаться по графу зависимостей, который вы получили, он же граф распределения, у блоков модулей много зависимостей. В этом и цель нашей работы: организовать команду так, чтобы параллельная разработка корректно применялась. Если вы понимаете, что ссылаетесь на другой модуль, то локально реализуйте заглушку, которая будет имитировать работу функции.<br>
Как работать <br>
1. При разработке модуля вы понимаете, что зависите от функций из других блоков. <br>
2. Временно вызывайте заглушку вместо реальной функции.<br>

Пример:
```c
/*
int COM_NN_D_stub(...) {
  return "...";
}
*/
```
